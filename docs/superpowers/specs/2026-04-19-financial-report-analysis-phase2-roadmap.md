# 财报分析 Phase-2 总纲

## 1. 目的

本文档定义 `financial-report-analysis` 在 Phase-1 完成后的下一阶段目标。

Phase-1 已完成：

- 独立 analysis service 与 HTTP contract
- `report/` forwarding 接入
- 基础 fact pipeline、质量闸门与 TTM
- 独立 ingestion path 的最小可用闭环
- 真实样本回归

Phase-2 不再围绕“是否独立服务”展开，而是聚焦于提升抽取底座，使服务从“单指标可用”推进到“可持续扩展的表格化抽取能力”。

## 2. Phase-2 总目标

在 **A 股 + 港股英文** 范围内，把当前以轻量文本正则为主的独立 ingestion path 升级为面向财务表格的结构化抽取底座。

Phase-2 的核心目标不是一次性覆盖全部字段，而是同时推进：

- 抽取准确率提升
- 核心字段覆盖率提升
- 真实样本回归稳定性提升

但三者的优先级必须建立在一个前提上：

**核心主表结构必须先稳定。**

## 3. 适用范围

### 3.1 市场与语言

Phase-2 继续只覆盖：

- A 股中文财报
- 港股英文财报

不纳入本阶段范围：

- 港股繁体中文主链路
- 美股
- 非财报类文档

### 3.2 报告类型

继续覆盖：

- 年报
- 中报 / 半年报
- 季报

### 3.3 服务边界

Phase-2 必须继续遵守当前架构边界：

- `financial-report-analysis` 是主分析服务
- `report/` 只承担 forwarding 与兼容入口
- 不允许把 `report/src/pdf_parser` 或旧 extractor 代码直接倒灌回独立服务

## 4. 成功标准

Phase-2 的完成标准不是“抽到更多字段”这么简单，而是以下三项同时成立：

1. 主表结构识别稳定
2. 期间、单位、币种、列语义稳定
3. 在此基础上，核心字段覆盖率持续提升

具体来说：

- 真实样本上的主表识别、列对齐、单位币种识别要可回归
- 不再依赖全文级脆弱匹配来判断 period / unit / currency
- 利润表及后续核心字段扩展建立在稳定的 table model 上，而不是继续堆字段级特例

## 5. 分阶段路线

Phase-2 拆为三个顺序执行的子阶段：

### 5.1 子阶段 1：表格结构抽取与对齐

目标：

- 识别主表与摘要表
- 提取表头中的期间、单位、币种、比较列语义
- 处理跨页续表
- 建立稳定的行列绑定
- 产出可复用的结构化 table model

这是 Phase-2 的地基。

### 5.2 子阶段 2：利润表深度抽取

目标：

- 在稳定 table model 上，把利润表从“可抽 revenue”扩展到“可抽一组核心利润指标”
- 把利润表做成 Phase-2 第一条完整主链路

建议优先字段：

- 营业收入
- 营业成本
- 毛利润 / 毛利率相关基础项
- 营业利润
- 净利润
- 扣非净利润

### 5.3 子阶段 3：三大报表核心字段扩展

目标：

- 把已验证的结构能力推广到三大主表
- 扩展资产负债表和现金流量表核心指标
- 让 `canonical_facts` 的核心覆盖更接近正式分析需要

建议优先字段：

- 总资产
- 总负债
- 归母权益 / 股东权益
- 经营活动现金流
- 投资活动现金流
- 筹资活动现金流

## 6. 优先级分层

并非所有表都以同优先级推进。

### 6.1 P0

P0 是三大主表：

- 利润表
- 资产负债表
- 现金流量表

它们构成主账本来源，是 Phase-2 的主验收轴。

### 6.2 P1

P1 是“主要财务数据表”等摘要型表格。

Phase-2 允许把这类表纳入结构识别与列语义抽取，但不要求与三大主表同等级推进，也不要求本阶段全部成为 canonical fact 主来源。

这样可以避免摘要表复杂度反向拖垮主表结构层设计。

## 7. 设计约束

Phase-2 必须明确避免以下反模式：

- 在不稳定的结构底座上继续扩字段
- 用全文关键词扫描替代表格局部上下文判断
- 在 `financial-report-analysis` 中重新复制旧 extractor 的大文件特例逻辑
- 让 LLM 成为主抽取链路

LLM 在 Phase-2 的位置应保持为辅助层，而不是主抽取器。可用场景包括：

- 表头语义补充
- 行标签模糊归一
- 冲突裁决时的辅助判别

但 period / unit / currency / value 的主提取链路仍应优先来自结构化规则。

## 8. 验收策略

Phase-2 必须以真实样本回归为主，不接受只靠 synthetic case 判完成。

建议最低样本矩阵：

- A 股年报样本
- A 股季报样本
- 港股英文季报样本
- 港股英文半年报或年报样本

每个子阶段的验收都必须回答：

- 主表是否被识别
- 期间是否被正确对齐
- 单位 / 币种是否来自局部有效上下文
- evidence 是否还能追溯
- quality gate 是否仍然有意义

## 9. 文档与计划策略

Phase-2 不建议一次性写完全部子 spec 再开始实现。

推荐流程：

1. 先写本总纲 spec
2. 只写子 spec 1：表格结构抽取与对齐
3. 子 spec 1 落地并收口后，再写子 spec 2 与子 spec 3

原因：

- 当前最大不确定性在表格结构层
- 子 spec 2/3 的大量细节依赖子 spec 1 的最终产物
- 过早写死后续字段设计，会增加返工概率

## 10. 结论

Phase-2 的正确方向不是继续堆字段规则，也不是直接把大模型变成 PDF 主解析器。

正确方向是：

- 先把表格结构抽取与对齐做稳
- 再让利润表形成第一条完整主链路
- 最后扩展到三大报表核心字段

这会让 `financial-report-analysis` 从“能工作的小闭环”升级成“可持续扩展的财报分析底座”。
